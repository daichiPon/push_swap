# Push_swap

> 2つのスタック(`a`と`b`)と、厳しく制限された操作だけを使って、ユニークな整数の
> スタックを最小手数で昇順に並べ替えるプロジェクト。アルゴリズムの計算量を
> 実際に体験しながら学ぶことが目的です。

## 説明/“Description”

**Push_swap** は42カリキュラムのアルゴリズム系プロジェクトです。

引数として渡されたユニークな整数のリストに対して、`push_swap` プログラムは
標準出力に、以下のものだけを使ってスタック `a` を昇順に並び替えるための
**最小限の操作列**を計算・出力します。

- 2つのスタック: `a`(初期状態で入力された数値が入る)と `b`(初期状態は空)
- 11種類の許可された操作: `sa`, `sb`, `ss`, `pa`, `pb`, `ra`, `rb`, `rr`,
  `rra`, `rrb`, `rrr`

アルゴリズムの計算量(complexity)を具体的に体験するため、本プログラムは
**実行時に切り替え可能な4種類のソート戦略**を実装しています。

| フラグ | 戦略 | 目標計算量 |
|---|---|---|
| `--simple` | ベースラインとなるアルゴリズム | O(n²) |
| `--medium` | チャンク/バケットベースのアルゴリズム | O(n√n) |
| `--complex` | 分割統治・基数ソート系のアルゴリズム | O(n log n) |
| `--adaptive`(デフォルト) | 入力の*disorder(乱雑度)*に応じて上記のいずれかを自動選択 | disorderに応じて O(n²) / O(n√n) / O(n log n) |

`--bench` モードを指定すると、`stderr` に入力の disorder(%)、使用した戦略名
とその理論的な計算量クラス、合計操作数、各操作の内訳が出力されます。

## 使い方/“Instructions”

### 必要環境

- Cコンパイラ(`cc`)
- `make`
- POSIX互換のシェル(以下の例は `bash`/`zsh` の記法を使用)

### コンパイル

```sh
make        # push_swap をビルド
make clean  # オブジェクトファイルを削除
make fclean # オブジェクトファイルとバイナリを削除
make re     # fclean + all
```

このMakefileは不要な再リンクを行いません。一部のソースファイルだけを変更して
`make` を再実行した場合、必要な部分だけがリビルド・再リンクされます。

出力は`srcs/utils/op_output.c` と `srcs/utils/bench_utils.c` に実装した最小限の
ヘルパー(`put_str_fd`、`put_nbr_fd`、`put_disorder_fd` など)で行っています。
リスト操作や文字列比較に必要な少数の補助関数(`ft_atoi`、`ft_strcmp`、
`ft_strncmp`、`ft_lstnew`、`ft_lstadd_back`)のみを `srcs/utils` 配下に
自前で持っています。

### 実行方法

```sh
./push_swap [戦略フラグ] N1 N2 N3 ...
```

- 戦略フラグは省略可能で、`--simple`、`--medium`、`--complex`、`--adaptive`
  のいずれかを指定できます。省略時は `--adaptive` が使われます。
- 引数は**重複のない整数**のリストで、`int` の範囲に収まる必要があります。
  別々の引数として渡しても、スペース区切りの1つの文字列として渡しても構いません。
- 引数が**何も渡されなかった場合**は、何も出力せずに終了します。
- 不正な入力(整数でない引数、範囲外の整数、重複した値など)の場合、
  `stderr` に `Error` を出力し、その後に `\n` を出力します。

#### 実行例

```sh
$> ./push_swap 2 1 3 6 5 8
ra
pb
rra
pb
pb
sa
pa
pa
pa
```

#### 操作数を数える

```sh
$> ARG="4 67 3 87 23"
$> ./push_swap --adaptive ${=ARG} | wc -l
17
```

### ベンチマークモード

```sh
$> ./push_swap --bench [フラグ] 1 2 ... 2> bench.txt
```

ベンチマークレポートは`--bench` を指定した場合のみ表示されます。

レポートに含まれる内容:

- 計算された disorder(小数点2桁の割合)
- 使用された戦略名とその理論的な計算量クラス
- 合計操作数
- 各操作種別(`sa`, `sb`, `ss`, `pa`, `pb`, `ra`, `rb`, `rr`, `rra`, `rrb`,
  `rrr`)の回数

例(100個のランダムな入力):

```
[bench] disorder: 44.53%
[bench] strategy: Adaptive / O(n√n)
[bench] total_ops: 828
[bench] sa: 0  sb: 0  ss: 0  pa: 100  pb: 100  
[bench] ra: 412  rb: 133  rr: 0  rra: 0  rrb: 83  rrr: 0  
```

戦略名の計算量クラスは出力上 ASCII で表記され、Simple は `O(n^2)`、
Medium は `O(n√n)`、Complex は `O(n log n)` と表示されます。

## 操作一覧

| 操作 | 効果 |
|---|---|
| `sa` | スタック `a` の先頭2要素を入れ替える(要素が1つ以下なら何もしない) |
| `sb` | スタック `b` の先頭2要素を入れ替える(要素が1つ以下なら何もしない) |
| `ss` | `sa` と `sb` を同時に行う |
| `pa` | `b` の先頭要素を `a` の先頭に移動する(`b` が空なら何もしない) |
| `pb` | `a` の先頭要素を `b` の先頭に移動する(`a` が空なら何もしない) |
| `ra` | `a` を1つ上にシフトし、先頭要素を末尾に移動する |
| `rb` | `b` を1つ上にシフトし、先頭要素を末尾に移動する |
| `rr` | `ra` と `rb` を同時に行う |
| `rra` | `a` を1つ下にシフトし、末尾要素を先頭に移動する |
| `rrb` | `b` を1つ下にシフトし、末尾要素を先頭に移動する |
| `rrr` | `rra` と `rrb` を同時に行う |

## Disorder(乱雑度)の指標

初期状態のスタック `a` の **disorder** は、`0`(既にソート済み)から
`1`(最悪の順序)までの値です。**いかなる操作も行う前に**、全てのペアに
対する「順序が逆転しているペア」の割合として計算されます。

```
disorder = (a[i] > a[j] となるペア (i, j), ただし i < j の数)
           / (n * (n - 1) / 2)
```

この値が `--adaptive` の戦略選択に使われます。

## アルゴリズム

### 1. Simple — 挿入ソート(Insertion Sort), O(n²)

**アルゴリズムの概要:**

スタック `a` の要素を1つずつ取り出し、すでに昇順に並んでいるスタック `b`
の正しい位置に挿入していく。`a` の全要素を `b` に移し終えたら、今度は `b`
から `a` へ1つずつ戻し、`a` 内でも正しい位置に挿入していくことで、最終的に
`a` 全体を昇順にソートする。

具体的な手順:

1. `a` の先頭要素(最大値)が `b` に挿入されるべき位置に来るよう、`a` を `ra`/`rra`
   で回転させる。
2. `pb` で要素を `a` から `b` に移す。これにより `b` は常に降順に保たれる。
3. `a` が3size以下になるまで 1〜2 を繰り返す。
4. `a`　の要素が3size以下のときにその中でソートを行う
5. `b` の要素を1つずつ `pa` で `a` に戻す。


**計算量の議論:**

- 要素数を `n` とする。
- フェーズ1(`a → b`): 各要素を `b` 内の正しい位置に挿入するために、
  最悪の場合 `b` を `O(n)` 回回転させる必要がある。これを `n` 個全てに
  対して行うため、最悪 `O(n²)` 回の操作。
- フェーズ2(`b → a`): 同様に、各要素を `a` 内の正しい位置に挿入する際、
  最悪 `O(n)` 回の回転が必要なため、こちらも最悪 `O(n²)`。
- したがって、全体の操作数は `O(n²) + O(n²) = O(n²)`。
- 入力がほぼソート済み(disorderが小さい)場合は、各要素の挿入位置探索に
  必要な回転数が少なくなるため、実際の操作数は `O(n²)` の最悪値より
  かなり小さくなる。これが `--adaptive` で「disorderが小さいときに
  Simpleを選ぶ」根拠の1つになる。

## 2. Medium — チャンクソート(Chunk Sort), O(n√n)

### アルゴリズムの概要:

`a` の各要素にあらかじめソート順位(0〜n-1 の index)を割り当て、
その順位の範囲 [0, n-1] を約 √n 個の「チャンク(区間)」に分割する。
チャンクはランク(値)が大きい方から順に処理し、そのチャンクに属する
要素を `a` から `b` へ移していく。各チャンクを `b` に移す際、チャンクの
中央値 mid を基準に、ランクが mid 以下の要素は `b` の最下部へ送り、
mid より大きい要素は `b` の上側に残すことで、`b` の中はチャンクごと・
上下半分ごとにグループ化された状態になる。全チャンクを処理した後、`b` の
中から最大ランクの要素を1つずつ探して `a` に戻すことで、`a` 全体を
昇順にソートする。

具体的な手順:

1. 各要素にソート順位(0〜n-1)を割り当て、chunk_size = ⌈√n⌉ として、
ランクの範囲 [0, n-1] を幅 chunk_size の約 √n 個のチャンクに
分割する。
2. ランクが最大のチャンク(最後のチャンク)から、ランク 0 を含む最初の
チャンクまで、順に以下を行う:

- `a` を `ra` で回転させながら走査し、現在のチャンク範囲 [min, max]
にランクが含まれる要素が見つかったら `pb` で `b` に移す。
`b` に移したばかりの要素のランクがそのチャンクの中央値
mid = (min + max) / 2 以下であれば、`rb` で `b` を回転させて
その要素を `b` の最下部へ送る。
- チャンク内の全要素(target = max - min + 1 個)が見つかるか、
a を1周しても新たに見つからなくなるまで繰り返す。



3. 全チャンクの処理が終わると `a` は空になり、`b` には全要素がチャンク
ごと・チャンク内の上下半分ごとにグループ化された状態で積まれている。
4. `b` が空になるまで、`b` の中で最大のランクを持つ要素の位置を探し、
その位置が `b` の上半分にあれば `rb`、下半分にあれば `rrb` で
回転させて先頭に移動させてから `pa` で `a`に戻す。最大ランクの要素
から順に `a`へ積まれていくため、最終的に `a`の先頭(top)が最小値、
底が最大値となる昇順ソート済みの状態になる。


計算量の議論:


チャンク数は √n 個。
フェーズ1(チャンク分配, 手順2): 1つのチャンクを処理する際、その時点
での `a`を1回走査するだけで該当ランクの要素を全て見つけられるため、
1チャンクあたりの操作数(`ra` + `pb` + `rb`)はその時点の `a` のサイズ
に比例して O(n)。これを √n 個のチャンク分繰り返すため、フェーズ1
全体では O(n) × O(√n) = O(n√n)。
フェーズ2(`b` → `a`, 手順4): find_max_pos 自体はC側のループであり
Push_swap操作には数えないが、見つけた要素を先頭まで移動させるための
rb/rrb の回数が問題になる。フェーズ1で `b` がチャンク・上下半分
ごとにグループ化されているため、各ステップで先頭まで移動させる回転数
は b のサイズに対して概ね O(√n) 程度に抑えられることを想定して
おり、n ステップ分で O(n√n) となる
したがって、全体の操作数はおおよそ O(n√n) + O(n√n) = O(n√n)。

### 3. Complex — 基数ソート(Radix Sort), O(n log n)

**アルゴリズムの概要:**

各要素の実際の値ではなく、あらかじめ割り当てられたソート順位(`index`、
`0`〜`n-1` の整数)に対してLSD(Least Significant Digit、最下位ビット優先)
基数ソートを適用する。最下位ビットから、`a` 内の最大ランク値を表現するのに
必要なビット数まで、各ビット位置について「ビットが `0` の要素を `b` へ、
`1` の要素は `a` に残す(回転で末尾側へ送る)」という分配操作を `a` の
全要素に対して1回行い、そのビット位置の処理が終わったら `b` に積まれた
要素をすべて `a` に戻す。これをビット数分繰り返すことでソートする。

具体的な手順:

1. `a` 内の要素の `index`(順位)の最大値(`= n-1`、`ft_max_value`)を求め、
   それを2進数で表すのに必要なビット数 `count_bit`(`ft_count_bit`)を
   求める。`count_bit ≈ ⌈log2(n)⌉`。
2. 最下位ビット(`i = 0`)から `count_bit - 1` まで、各ビット位置 `i` に
   ついて以下を行う:
   - その時点の `a` の要素数 `size` を記録する。
   - `size` 回繰り返し、`a` の先頭要素について `index` の bit `i`
     (`a->top->index >> i & 1`)を確認する。
     - bit `i` が `0` であれば `pb` で `b` に移す。
     - bit `i` が `1` であれば `ra` で `a` を回転させ、要素を相対順序を
       保ったまま `a` の末尾側へ送る。
   - `size` 回の処理(= このビット位置についての `a` 全要素の1周分)が
     終わったら、`b` が空になるまで `pa` で `b` の要素をすべて `a` に
     戻す(`push_btoa`)。
3. 全ビット位置について処理を終えると、`a` は昇順にソートされている。

**計算量の議論:**

- 必要なビット数は `count_bit ≈ ⌈log2(n)⌉`。
- 各ビット位置の処理では、その時点の `a` の全要素(`n` 個)に対して
  1回ずつ「`pb` または `ra`」を行い、さらに `b` に移った要素を `pa` で
  `a` に戻すため、1ビット位置あたり `O(n)` 回の操作。
- これを `count_bit ≈ O(log n)` 回繰り返すため、全体の操作数は
  `O(n) × O(log n) = O(n log n)`。
- 入力のdisorderが大きい(ほぼランダムに近い)場合でも操作数が
  `O(n log n)` で頭打ちになるため、`--adaptive` では「disorderが大きい
  ときにComplexを選ぶ」根拠となる。

### 4. Adaptive

adaptive戦略は入力のdisorderを計測し、以下の基準に従って上記いずれかの
戦略に振り分けます。

| Disorder | 使用される戦略 | 目標計算量 |
|---|---|---|
| `< 0.20` | Simple | O(n²) |
| `0.20 ≤ disorder < 0.50` | Medium | O(n√n) |
| `≥ 0.50` | Complex | O(n log n) |

**計算量の議論(Push_swapモデル内):**

各レジームでの時間計算量(=生成される操作数)の上限は、対応する戦略の
セクションで示した通りである。

| Disorder | 使用される戦略 | 時間(操作数)の上限 | 追加メモリ(空間) |
|---|---|---|---|
| `< 0.20` | Simple(挿入ソート) | `O(n²)` (フェーズ1・2それぞれ `O(n²)`) | `O(n)`(全要素を保持するスタック分のみ) |
| `0.20 ≤ disorder < 0.50` | Medium(チャンクソート) | `O(n√n)` (チャンク分配 `O(n√n)` + 挿入 `O(n√n)` + 戻し `O(n)`) | `O(n)`(全要素分のスタック + チャンク境界値を保持する `O(√n)` の補助配列) |
| `≥ 0.50` | Complex(基数ソート) | `O(n log n)` (ビット数 `⌈log2 n⌉` 回 × 各回 `O(n)`) | `O(n)`(全要素分のスタック + 各要素の順位を保持する `O(n)` の補助配列) |

いずれのレジームでも、追加メモリは入力サイズ `n` に対して線形(`O(n)`)で
収まっており、スタック自体に格納する要素数を超える大きな補助構造は
必要としない。

## パフォーマンス

subjectで定義された目標値(`checker` で検証):

| 入力サイズ | 合格 | 良好 | 優秀 |
|---|---|---|---|
| 100個のランダムな数 | 2000操作未満 | 1500操作未満 | 700操作未満 |
| 500個のランダムな数 | 12000操作未満 | 8000操作未満 | 5500操作未満 |

| 入力サイズ | 使用戦略 | 平均操作数 |
|---|---|---|
| 100 | adaptive(disorder ≈ 0.5 → Complex) | 約 930 |
| 500 | adaptive(disorder ≈ 0.5 → Complex) | 約 7300 |

> ランダムに生成した値(`shuf`)は disorder がおよそ 0.5 になるため、
> `--adaptive` では主に Complex(基数ソート)が選択されます。上記は
> 各サイズ 10 回ずつ実行した操作数の平均値です。

再現方法の例:

```sh
ARG="4 67 3 87 23" && \
./push_swap --complex ${=ARG} | \

```


## エラー処理

```sh
$> ./push_swap --adaptive 0 one 2 3
Error
$> ./push_swap --simple 3 2 3
Error
```

エラーは `stderr` に `Error\n` として出力され、以下を含みます(これらに
限定されません)。

- 数値ではない引数
- `int` の範囲を超える整数
- 重複した値
- (`checker` の場合)`stdin` から読み込んだ未知または不正な形式の命令
